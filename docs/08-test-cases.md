# 08 — 手动测试用例

## 测试环境

- **数据库：** MySQL 8（Docker 容器 `mysql`，端口 3306）
- **种子数据：** `npm run prisma:seed` 后重置
- **启动命令：** `npm run dev` → http://localhost:3000
- **种子账号：**

| 角色 | 邮箱 | 密码 |
|------|------|------|
| ADMIN | admin@example.com | admin123 |
| USER | user@example.com | user123 |

## 推荐测试顺序

1. 环境与启动（确保应用可访问）
2. 注册 / 登录 / 退出（认证是其他功能的前提）
3. 权限与导航（验证 proxy.ts + Header 行为）
4. 商品浏览（无需登录的核心功能）
5. 购物车（需登录，依赖商品）
6. 下单（依赖购物车 + 商品库存）
7. 订单列表 / 订单详情
8. 模拟支付
9. 取消订单与库存恢复
10. 后台管理（需 ADMIN 登录）
11. 商品管理（依赖后台）
12. 分类管理
13. 订单状态变更
14. loading / empty / error 状态

> **说明：** "未测" = 尚未执行，"通过" = 行为符合预期，"失败" = 行为异常需修复。
