# 05 — 测试与发布策略

## 一、测试分层

### P0 冒烟测试（18 条，20~30 分钟）

每次合并到 `main` 前或部署后执行。详见 [冒烟测试清单](#po-冒烟测试清单)。

### 完整回归测试（76 条，约 2 小时）

详见 [完整测试用例](#完整测试用例)。大版本发布前执行全部，功能分支合并前选取关联模块执行。

自由操作，发现用例未覆盖的边界问题：异常输入、快速连续点击、浏览器前进后退、多标签页、Cookie 清除、直接输入路由。

## 五、发布策略

### 分支工作流（Trunk-Based Development 简化版）

```
main ← feature/auth, feature/products, feature/cart, feature/orders, feature/admin, fix/*
```

规则：`main` 始终可构建；功能分支开发；合并前验证。

### 合并前检查

| # | 检查项 | 命令 |
|---|--------|------|
| 1 | Prisma Schema | `npm run prisma:validate` |
| 2 | TypeScript | `npx tsc --noEmit` |
| 3 | Build | `npm run build` |
| 4 | Lint | `npm run lint`（0 errors；如未配置则以 tsc+build 为准） |
| 5 | Clean tree | `git status --short` 无输出 |

### 最低发布标准

1. lint 0 errors / tsc 零错误 / build 成功
2. 18 条冒烟测试全部通过
3. 所有 feature 分支已合并
4. 文档齐全

### Staging 环境设计

Vercel Preview Deployment → 独立 staging MySQL → 冒烟测试 → 探索测试 → 手动触发生产部署

### Vercel Serverless + MySQL 注意事项

Serverless 函数按需启动，高并发时可能耗尽 MySQL 连接数。方案：
- **推荐：** Prisma Accelerate 或托管连接池
- **备选：** 中间件连接池（如 PgBouncer for MySQL）
- **简易：** adapter `connectionLimit` 参数限制

Mini Mall 当前开发阶段直连无恙，部署前需评估。

### 灰度发布方案（设计）

- 蓝绿部署：Blue(当前) + Green(新版) → 验证通过后切换
- 金丝雀：5% → 25% → 50% → 100%，异常即回滚

### 回滚策略

- **代码：** `git revert` 或 Vercel `rollback`
- **数据库：** 每个 migration 预先准备 rollback SQL；破坏性迁移先部署兼容代码再执行；`mysqldump` 每日备份
- **注意：** Prisma 不提供自动回滚命令，需手动准备和执行回滚脚本

### 监控指标

| 指标 | 获取方式 |
|------|---------|
| 错误率、响应时间 | Vercel Analytics |
| 下单成功率 | DB 查询 |
| 库存异常（stock < 0） | DB 约束 |
| 日订单量 / 日销售额 | SQL 聚合查询 |
