# 01 — 项目定位与功能边界

## 一句话定位

一个可以跑起来、讲得清、代码干净的微型电商系统，适合在简历中展示全栈能力和在面试中讲解架构决策。

## 技术栈

| 层 | 技术 |
|----|------|
| 框架 | Next.js 16 |
| 语言 | TypeScript (strict) |
| 数据库 | MySQL 8 |
| ORM | Prisma latest / 6+ |
| 样式 | Tailwind CSS 4 |
| 认证 | JWT + httpOnly Cookie（jose 库） |
| 密码 | bcryptjs |
| 校验 | Zod |

## 目标受众

- 面试官（通过代码和架构展示能力）
- 自己（作为后续项目的参考模板）

## 功能边界

### 前台（Customer-facing）

| 功能 | 描述 |
|------|------|
| 商品列表 | 分页浏览、按分类筛选、关键词搜索、排序 |
| 商品详情 | 图片展示、描述、价格、库存、加入购物车 |
| 用户注册 | 邮箱 + 密码 + 昵称，注册即登录（无邮箱验证） |
| 用户登录 | JWT + httpOnly Cookie，7 天有效期 |
| 用户退出 | 清除 Cookie，重定向首页 |
| 购物车 | 查看、修改数量、删除商品 |
| 创建订单 | 填写收货信息 → 下单（事务 + 库存扣减 + 商品快照） |
| 我的订单 | 订单列表 + 订单详情 + 模拟支付 |

### 后台（Admin，需 ADMIN 角色）

| 功能 | 描述 |
|------|------|
| Dashboard | 商品总数、已上架商品数、订单总数、待处理订单数、总销售额、用户总数 |
| 商品管理 | 列表、新增、编辑、上架/下架（不支持真删除） |
| 分类管理 | 列表、新增、编辑（不支持删除） |
| 订单管理 | 所有订单列表、订单详情、状态变更 |

### 明确不做的功能

- 商品真删除（仅支持下架）
- 分类删除
- 用户地址簿（下单时直接填写收货信息）
- 商品评价 / 收藏
- 优惠券 / 促销
- 真实支付（仅模拟 PENDING → PAID）
- 物流追踪
- 图片上传（仅 URL 输入，最多 6 张）
- 再次购买
- 邮箱验证
- API Routes（后续扩展）
- i18n / 多语言
